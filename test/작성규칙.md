# 테스트 작성 규칙



## 1. 사용처 관점의 테스트
- 사용한 방식을 기준으로 작성한다.
- 제목 규칙 : - 관점명 : 관련메소드 및 설명

## 2. 구조 관점의 테스트
- 클래스 및 메소드 또는 함수의 관점에서 테스트를 작성한다.
- 파레메터의 가능한 조건을 최대한 나열한다.
- 제목 규칙 : - 메소드명 | 함수명 : 설명

## 3. 커비리지 관점의 테스트
- Uncovered Line 관점의 테스트 코드를 작성한다.
- 사용환경에 대한 테스트 코드를 작성한다.
    + node, web
    + ES5, ES6, ES6+
- 제목 규칙 : - ETC : 설명 또는 메소드
